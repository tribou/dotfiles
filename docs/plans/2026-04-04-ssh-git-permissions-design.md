# SSH Git Permissions Design

**Date:** 2026-04-04
**Branch:** ssh-git-permissions

## Overview

A setup for dev Linux servers where LLM agent coding tools run under a separate Linux user (`agent`) with isolated git credentials, while the main user (`tribou`) retains full git access. Provides revocation, per-repo access control, and a clean audit trail.

## Architecture

```
Your laptop
    └── ssh tribou@server
            ├── tribou (full git access via personal SSH key)
            │     └── sudo -u agent -i  →  alias: llm
            └── agent (limited git access via deploy key)
                  └── ~/dev/ → shared repos (group-write via devteam group)
```

Both users belong to a `devteam` Linux group with group-write access to `~/dev/`. Each user has their own `~/.ssh/config` pointing to their respective SSH key. Git commits from `agent` carry a distinct identity, making them visually distinguishable in history.

Deploy keys are registered per-repo on GitHub/GitLab — the agent only has access to repos you explicitly opt in, and access can be revoked from the GitHub UI at any time.

## Components

| Piece | Location |
|---|---|
| Setup script | `agent/setup-user.sh` |
| Agent overrides | `agent/overrides.sh` |
| Convenience alias | `lib/commands.sh` (`alias llm=...`) |
| Deploy key | Registered manually on GitHub per-repo |

## Setup Script

`agent/setup-user.sh` — run once on any server where agent support is needed. Idempotent (safe to re-run).

Steps performed:
1. Create `agent` Linux user
2. Create `devteam` group, add `tribou` and `agent`
3. Set group ownership + `setgid` on `~/dev/` (new files inherit group automatically)
4. Generate `agent`'s SSH key at `/home/agent/.ssh/id_ed25519` (if not already present)
5. Write `agent`'s `~/.ssh/config`
6. Write `agent`'s `~/.bash_profile` sourcing dotfiles (path resolved at setup time) then `~/.agent_overrides.sh`
7. Symlink `~agent/.agent_overrides.sh` → `$DOTFILES/agent/overrides.sh`
8. Write `agent`'s `~/.gitconfig`
9. Add passwordless sudo entry: `tribou ALL=(agent) NOPASSWD: /bin/bash`
10. Print the public key for manual registration as a GitHub/GitLab deploy key

## User & Filesystem Setup

```bash
sudo useradd -m -s /bin/bash agent
sudo groupadd devteam
sudo usermod -aG devteam tribou
sudo usermod -aG devteam agent
sudo chown -R tribou:devteam ~/dev/
sudo chmod -R g+rw ~/dev/
sudo find ~/dev/ -type d -exec chmod g+s {} \;
```

## Git Credential Isolation

Agent's SSH key (no passphrase — agents can't type one):
```bash
sudo -u agent ssh-keygen -t ed25519 -f /home/agent/.ssh/id_ed25519 -C "agent@yourserver"
```

Agent's `~/.ssh/config`:
```
Host github.com
  IdentityFile ~/.ssh/id_ed25519
  IdentitiesOnly yes
```

Agent's `~/.gitconfig`:
```ini
[user]
  name = Agent
  email = tribou@users.noreply.github.com
```

Git identity is also set via env vars in `agent/overrides.sh` to ensure it takes precedence regardless of repo-level config.

## Agent Environment

Agent's `~/.bash_profile` (generated by setup script with resolved `$DOTFILES` path):
```bash
source /home/tribou/dev/dotfiles/bash_profile
source ~/.agent_overrides.sh
```

`agent/overrides.sh` (version-controlled in dotfiles, symlinked into agent's home):
```bash
# Git identity overrides
export GIT_AUTHOR_NAME="Agent"
export GIT_COMMITTER_NAME="Agent"
export GIT_AUTHOR_EMAIL="tribou@users.noreply.github.com"
export GIT_COMMITTER_EMAIL="tribou@users.noreply.github.com"

# Prompt — [llm] prefix on existing PS1 style
PS1='\[\033[0;34m\][llm] \W $(get_git_location) > \[\]'
```

Agent shares all dotfiles config: bash functions, neovim, tmux, aliases.

## Workflow

```bash
ssh tribou@server   # normal login
llm                 # switch to agent user (alias for: sudo -u agent -i)
# launch claude, aider, cursor, etc.
```

## Revocation

- **Per-repo**: Remove deploy key from GitHub/GitLab UI — immediate, no server changes
- **Full lockout**: `sudo passwd -l agent` — locks the Linux account
- Both can be done independently

## Audit Trail

Three layers — no extra setup required:

1. **Git history**: Agent commits show `Agent <tribou@users.noreply.github.com>` — distinct from your normal commits
2. **GitHub deploy key logs**: Push activity attributed to the deploy key, visible under repo Settings → Deploy keys
3. **Linux system logs**: Commands run under `agent` user appear in `/var/log/auth.log` and process listings
